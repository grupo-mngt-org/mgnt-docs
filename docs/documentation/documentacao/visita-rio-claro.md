# ***RESUMO:***

## FINANCEIRO:

- CONTAS A PAGAR E RECEBER
    - Eles necessitam da centralização desses dados já que eles estão em sistema distintos
- DRE: Demonstração de resultado independente se foi pago ou não
    - Segue a mesma linha do contas a pagar e receber
    - Temos no Omie e no Sienge
    - Omie → FInanceiro → Relatórios → Data de emissão
    - Sienge → Suporte/Decisão → Gerencial financeiro → Analitico de apropriação (comprometido) → Data de emissão
- Omie não tem integração com prefeitura
    - Ela precisa gerar a nota na prefeitura
    - Para depois dela “deferida” vai para Omie
- Consultar notas emitidas (A10/Mais Armazem/Energia/Centro/MNGT)
    - Atualmente ela precisa consultar uma a uma manualmente

## FAROL (COMPRAS E PLANEJAMENTO) :

- [COMPRAS] Que seria dar visibilidade a prioridade de compra a partir da solicitação da obra

## **MAIS ARMAZEM:**

- A maior problemática é o ERP UNO que não nos permite usa-lo para integrações
- Faturamento feito manualmente pois o ERP não registra dados importantes da operação (ex: quantidade de paletes in)
- CRONO análise
    - Eles tem uma planilha com as informação de entrada e saida. Nela eu tenho informações sobre o caminha, horário de entrada, tempo de carregamento/descarregamento. Assim, a partir dela poderemos gerar indicadores
    - Planilhas de faturamente para consegui calcular a partir das especificidades o faturamento das operações
    - ZOHO -  sistema usado na operação da OJI PAPÉIS, onde a partir do coletor é registrado o palete que o serviço foi feito e assim enviado, do ZOHO para uma planilha e está integrada no SAP da OJI
        - A ideia é expandir essa coleta para as outras operações e podermos registrar esses serviços para que possa ser geradaa cobrança dos mesmos
